Requirements

Functional Requirements

  • User can display and filter products on the system without the logging in
  • User can sign in the system
  • User can log in the system
  • User can edit his/her profile
  • User can delete his/her profile
  • User can add product and describe product's information
  • User can edit product's information
  • User can create an auction
  • User can delete an auction until the last day
  • User can display and filter auctions on the system
  • User can join an auction
  • User can rate other users
  • User can send message to the other users
  • User can display other users' profiles
  • User can click 'sold' button when he/she deals with other user
  • User can log out

Non-Functional Requirements

  • Open Source:

    Our project will be certified with GPL

  • Scalability:

    System will run independent from increasing number of users

  • Availability:

    The system can available 7/24 under any situation

  • Security:

    Users' passwords will store in database as hash

  • Usability:

    System will look like other common websites thanks to it, its usage is very simple

  • Response Time:

    At auction system will response any request under 1 second
